# Extended Use Case Description
### Use Case: _Add Brew Results_

### Actors: _Registered User_

### Description:
1. The user navigates to the Brew page after making a recipe
2. The user selects which coffee and recipe they used
3. The user rates the outcome of the brew on a variety of numeric scales and submits
3. After submitting, the brew result is stored in a database and is fetched on the stats page

### Cross Ref:
- Functional Requirements: 1 and 3
- Nonfunctional Requirements: 3, 4, and 5

### Requisite Use Cases: 
- User must have registered and logged in
